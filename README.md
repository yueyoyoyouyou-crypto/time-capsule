# 🪐 时光寄存 · STAR MEMORIAL

> 爱 — 是唯一能穿越时空的坐标

一款纪念逝去亲人、爱宠的微信小程序 Demo。将思念化为星球上的星星，通过规则引擎驱动的「人设对话」跨越时空诉说想念。

---

## ✨ 功能

| 模块 | 说明 |
|------|------|
| **🏠 首页** | 每日寄语轮换 / 特别日子倒计时提醒 |
| **🪐 星球页** | 自转地球 + 轨道头像动画；轨道与名片双向联动定位 |
| **💬 时空对话机** | 基于角色身份、喜好、语境的规则引擎自动回复；支持亲人/爱宠两套角色系统 |
| **📖 回忆录** | 时间线式回忆管理，支持日期、标签、emoji 贴图 |
| **📔 我的笔记本** | 私密笔记，仅自己可见，emoji + 标题 + 内容 |
| **👤 我的** | 个人资料、数据统计（守护天数/对话次数）、月度星愿进度 |
| **☁️ 备份恢复** | 一键导出/导入 JSON 备份文件 |

---

## 🚀 如何运行

### 方式一：直接打开（功能完整）

```bash
open index.html
```

本地直接双击 `index.html` 即可运行（所有数据存储在浏览器 localStorage）。

### 方式二：本地服务器（推荐）

```bash
python3 server.py
```

浏览器打开 [http://127.0.0.1:8899](http://127.0.0.1:8899)。

> 端口 8899，如被占用可在 `server.py` 中修改 `PORT` 变量。

## 🧠 对话引擎原理

对话机不调用任何 AI 接口，完全由**本地规则引擎**驱动：

1. **角色建模** — 根据「关系」自动推断角色风格（奶奶叫你"乖孙儿"、爸妈叫你"宝贝"、爱宠用"汪汪/喵呜"语气）
2. **语境匹配** — 正则匹配用户输入（想念、吃饭、做梦、难过、生日、天气…），返回贴合身份的回复
3. **喜好引用** — 配置中填写的「TA 喜欢的」会在对话中被自动提及
4. **爱宠专属** — 宠物有一套独立的拟人化回复逻辑

## 🛠 技术栈

- 纯前端（HTML + CSS + JavaScript），零依赖
- CSS 变量主题系统（暖纸米色 / 深空双套色系）
- 纯 CSS 动画（地球自转、云层漂移、轨道运行、星星闪烁、流星划过）
- localStorage 结构化持久化（v2 数据格式）
- 规则引擎驱动的角色扮演对话
- 微信胶囊/手机框 UI 模拟

## 📝 学了什么

1. **纯 CSS 行星动画** — 多层轨道、自转地图（双份首尾相接无缝平移）、反向云层、box-shadow 模拟大气辉光
2. **规则引擎设计** — 用正则 + 角色模型构建类人对话，不依赖 API
3. **单文件架构** — 在 ~1900 行单 HTML 中组织路由、数据层、UI 层、对话引擎，不引入框架
4. **localStorage 版本管理** — v2 数据格式、跨月星愿归零、备份恢复兼容性检查
5. **小程序 UI 模拟** — 微信胶囊、TabBar、底部抽屉 Sheet、手机框适配
